### Промпт для разработки веб‑приложения HIIT‑таймера на React с TypeScript


**Цель:** создать веб‑приложение для HIIT‑тренировок *SporThick.HIIT-o-Maniac*, оптимизированное для использования на мобильных устройствах (в т. ч. на iPhone через браузер), с фокусом на простоту интерфейса, производительность и автономную работу. Приложение будет опубликовано на GitHub Pages.


#### 1. Технологический стек

* **Фреймворк:** React 19+ с Vite и TypeScript (быстрая сборка и горячая перезагрузка).
* **Язык программирования:** TypeScript 6+ (статическая типизация, улучшенная поддержка IDE).
* **Управление состоянием:** Zustand (лёгковесное решение для несложного приложения, с полной поддержкой TypeScript).
* **Маршрутизация:** React Router Dom для навигации между экранами.
* **Стилизация:** Tailwind CSS (адаптивный дизайн «из коробки», быстрая вёрстка).
* **Хранение данных:** LocalStorage (для сохранения тренировок и настроек пользователя).
* **Оффлайн‑работа:** Vite PWA Plugin + Service Worker (кэширование ресурсов и работа без интернета).
* **Сборка и деплой:** Vite + GitHub Actions (автоматическая сборка и деплой на GitHub Pages).
* **Дизайн:** Mobile‑First подход, соблюдение принципов Material Design (интуитивно понятный интерфейс для касаний).

#### 2. Типы TypeScript


Определите интерфейсы для строгой типизации данных:

```typescript
interface Exercise {
  id: string;
  name: string;
  duration: number; // в секундах
}

interface Workout {
  id: string;
  name: string;
  exercises: Exercise[];
  rounds: number; // 0 = бесконечность
}

interface AppState {
  workouts: Workout[];
  settings: {
    soundEnabled: boolean;
  };
}
```

#### 3. Структура данных

Формат хранения в LocalStorage:
```json
{
  "workouts": [
    {
      "id": "uuid-v4",
      "name": "Моя тренировка",
      "exercises": [
        {
          "id": "uuid-v4",
          "name": "Приседания",
          "duration": 30
        }
      ],
      "rounds": 3
    }
  ],
  "settings": {
    "soundEnabled": true
  }
}
```

**Правила работы с ID:**
* Генерировать через `crypto.randomUUID()`.
* Перед сохранением проверять уникальность ID через поиск по массиву.
* При импорте/экспорте — валидировать UUID‑формат, при ошибке — переназначать ID.

#### 4. Функциональные требования

##### Главный экран
* **Хеде́р:**
    * счётчик тренировок («Тренировок $N$»);
    * кнопка «+» (добавление новой тренировки) в правом верхнем углу.
* **Основной контент:**
    * список сохранённых тренировок с возможностью перетаскивания (drag‑and‑drop) для изменения порядка;
    * строка поиска над списком (фильтрует по названию тренировки и упражнения);
    * если список пуст — центрированное сообщение «Тренировок пока нет. Нажмите „+“, чтобы создать первую!» и крупная кнопка «+».
* **Действия на тренировке (по свайпу влево или через меню «…»):**
    * «Посмотреть» → попап с детализацией упражнений;
    * «Начать» → переход на экран тренировки;
    * «Редактировать» → переход на экран редактирования;
    * «Удалить» → диалоговое окно с подтверждением («Вы уверены, что хотите удалить эту тренировку?» и кнопки «Отмена»/«Удалить»).

##### Экран создания/редактирования тренировки
* **Хеде́р:** поле для ввода названия тренировки (placeholder «Новая тренировка»).
* **Основной контент:**
    * список упражнений (можно менять порядок перетаскиванием);
    * если упражнений нет — сообщение «Упражнений пока нет. Нажмите „+“, чтобы добавить первое» и кнопка «+»;
    * карточка упражнения:
        * поле для редактирования названия (по умолчанию «Упр. $1$», «Упр. $2$» и т. д.);
        * поле ввода времени в секундах (spinner или клавиатура с ограничением на ввод цифр);
        * кнопка удаления упражнения (иконка корзины).
* **Настройки тренировки:**
    * поле «Количество кругов» (spinner) от $1$ до $\infty$ (в интерфейсе обозначить как «1+»);
    * по умолчанию — $1$ круг.
* **Футер:** кнопки «Сохранить» и «Отмена».

##### Экран тренировки (общий для всех устройств)
* **Отображаемая информация:**
    * крупный обратный отсчёт таймера текущего упражнения (цифры занимают 70 % ширины экрана);
    * прогресс‑бар под цифрами (заполняется по мере отсчёта);
    * цвет цифр меняется: зелёный (больше $10$ с), жёлтый ($5$–$10$ с), красный (меньше $5$ с);
    * статус «Круг $X$ из $Y$» (если $Y > 1$ или $Y = \infty$);
    * статус «Упражнение $X$ из $Y$» (сбрасывается при начале нового круга);
    * название текущего упражнения.
* **Управление:**
    * крупные кнопки «Старт/Пауза», «Пропустить» (переход к следующему упражнению/кругу), «Стоп» (выход с подтверждением);
    * автоматическая смена упражнения по завершении таймера;
    * автоматический переход к следующему кругу по завершении всех упражнений в круге;
    * звуковые сигналы:
        * старт упражнения — короткий высокий сигнал;
        * конец упражнения — средний сигнал;
        * конец круга — длинный низкий сигнал;
        * завершение всей тренировки — двойной сигнал.
* **После завершения:** экран «Тренировка завершена!» с:
    * общим временем;
    * количеством кругов;
    * кнопкой «Вернуться к списку».
* **Адаптивность:**
    * **Портретная ориентация:** основной режим.
    * **Альбомная ориентация:**
        * таймер занимает весь экран;
        * кнопки «Старт/Пауза», «Пропустить», «Стоп» — внизу;
        * статус («Круг $X$ из $Y$») — вверху.
    * **Планшеты:** увеличить размер кнопок и шрифтов на $20$ %.

#### 5. Логика работы таймера

* **Фон:** таймер продолжает отсчёт при заблокированном экране (используем `Page Visibility API` для паузы при сворачивании вкладки).
* **Смена упражнения:** автоматически по истечении времени (без задержки).
* **Пропуск упражнения:** без подтверждения (одно нажатие кнопки).
* **Конец тренировки:** остановка таймера + экран завершения (не зацикливать).
* **Закрытие вкладки/браузера:**
    * сохранять прогресс тренировки в LocalStorage каждые $2$ с;
    * при повторном открытии — предлагать «Продолжить тренировку?» с кнопками «Да»/«Отмена».

#### 6. PWA‑специфика
* **Иконки:** 192×192 px и 512×512 px (PNG, прозрачный фон).
* **Цвет темы:** `#10b981` (бирюзовый).
* **Режим отображения:** `standalone` (полноэкранный, без адресной строки).
* **Сообщение об установке:** баннер «Добавьте SporThick.HIIT-o-Maniac на экран „Домой“ для удобства» (показывается один раз после 2‑го открытия).

#### 7. Обработка ошибок
* **LocalStorage недоступен:**
    * показывать попап «Данные не могут быть сохранены. Включите cookies и LocalStorage в настройках браузера».
    * разрешить использовать приложение в режиме «только просмотр» (без сохранения).
* **Service Worker не регистрируется:**
    * продолжать работу в онлайн‑режиме;
    * показать уведомление «Оффлайн‑режим недоступен. Проверьте интернет‑соединение».
* **Повреждение данных:**
    * при загрузке проверять JSON на валидность;
    * если ошибка — предлагать восстановить из последнего бэкапа или сбросить данные.

#### 8. Деплой и CI/CD
* **Репозиторий:** GitHub (имя: `SporThick.HIIT-o-Maniac`).
* **Ветвь для деплоя:** `main`.
* **Автоматизация:** GitHub Actions для сборки (Vite) и публикации на GitHub Pages.
* **Очистка кэша:** при деплое новой версии — принудительное обновление Service Worker и кэша.

#### 9. Ключевые сценарии тестирования
1. Установка PWA на iPhone (Safari → «Поделиться» → «На экран „Домой“»).
2. Работа таймера при заблокированном экране.
3. Drag‑and‑drop на сенсорных устройствах.
4. Фильтрация по названию упражнения.
5. Восстановление тренировки после закрытия браузера.
6. Смена ориентации экрана во время отсчёта.
7. Звуковые сигналы на iPhone (проверка совместимости).
8. Сохранение прогресса каждые $2$ с в LocalStorage.
9. Проверка корректности TypeScript‑типов во время разработки и сборки.
