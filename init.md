## Промпт для разработки HIIT‑таймера (iOS + watchOS) с Bluetooth‑синхронизацией

Цель создать кросс‑платформенное приложение для HIIT‑тренировок с синхронизацией данных через Bluetooth (WatchConnectivity), ориентированное на простоту использования и энергоэффективность.

### 1. Технологический стек

 Язык Swift 5.7+.
 UI‑фреймворк SwiftUI (унифицированный интерфейс для iOS и watchOS).
 Синхронизация WatchConnectivity (передача данных между iPhone и Apple Watch через Bluetooth).
 Хранение данных
     iOS UserDefaults (для простых данных) или Core Data (если планируется расширение функционала);
     watchOS UserDefaults.
 Реактивное программирование Combine (для обработки событий и обновлений UI).
 Дизайн соблюдение Human Interface Guidelines (HIG) для iOS и watchOS.

### 2. Функциональные требования

#### Главный экран (iOS)
 Хеде́р
     счётчик тренировок («Тренировок $N$»);
     кнопка «+» (добавление новой тренировки) в правом верхнем углу.
 Основной контент
     список сохранённых тренировок с возможностью перетаскивания (drag‑and‑drop) для изменения порядка;
     если список пуст центрированное сообщение «Тренировок пока нет. Нажмите „+“, чтобы создать первую!» и крупная кнопка «+».
 Действия на тренировке (по свайпу влево или через меню «…»)
     «Посмотреть» → попап с детализацией упражнений;
     «Начать» → переход на экран тренировки;
     «Редактировать» → переход на экран редактирования;
     «Удалить» → диалоговое окно с подтверждением («Вы уверены, что хотите удалить эту тренировку» и кнопки «Отмена»«Удалить»).

#### Экран созданияредактирования тренировки (iOS)
 Хеде́р поле для ввода названия тренировки (placeholder «Новая тренировка»).
 Основной контент
     список упражнений (можно менять порядок перетаскиванием);
     если упражнений нет сообщение «Упражнений пока нет. Нажмите „+“, чтобы добавить первое» и кнопка «+»;
     карточка упражнения
         поле для редактирования названия (по умолчанию «Упр. $1$», «Упр. $2$» и т. д.);
         поле ввода времени в секундах (spinner или клавиатура с ограничением на ввод цифр);
         кнопка удаления упражнения (иконка корзины).
 Настройки тренировки
     поле «Количество кругов» (spinner) от $1$ до $infty$ (в интерфейсе можно обозначить как «1+»);
     по умолчанию $1$ круг.
 Футер кнопки «Сохранить» и «Отмена».

#### Главный экран (watchOS)
 Контент
     компактный список тренировок (названия и, опционально, количество упражненийкругов);
     если тренировок нет сообщение «Создайте тренировки на iPhone»;
     отсутствие кнопки «+» (подчёркивает зависимость от iOS‑версии).
 Действия при выборе тренировки
     «Посмотреть» → попаплист с перечнем упражнений и их длительностью;
     «Начать» → переход на экран тренировки.

#### Экран тренировки (общий для iOS и watchOS)
 Отображаемая информация
     крупный обратный отсчёт таймера текущего упражнения (цифры должны быть хорошо читаемы);
     статус «Круг $X$ из $Y$» (если $Y  1$ или $Y = infty$);
     статус «Упражнение $X$ из $Y$» (сбрасывается при начале нового круга);
     название текущего упражнения.
 Управление
     крупные кнопки «СтартПауза», «Пропустить» (переход к следующему упражнениюкругу), «Стоп» (выход с подтверждением);
     автоматическая смена упражнения по завершении таймера;
     автоматический переход к следующему кругу по завершении всех упражнений в круге;
     визуальная и тактильная (haptic) обратная связь при смене упражнениякруга;
     опционально звуковой сигнал в началеконце упражнения.
 Дизайн (watchOS)
     минимум текста, максимум иконок и крупных цифр;
     использование Digital Crown для быстрой перемотки по упражнениям (если нужно).

### 3. Реализация синхронизации через WatchConnectivity

Шаг 1. Инициализация сессии

В `AppDelegate` (iOS) и `ExtensionDelegate` (watchOS) активируем сессию

```swift
import WatchConnectivity

class AppDelegate UIResponder, UIApplicationDelegate {
    func application(_ application UIApplication, didFinishLaunchingWithOptions launchOptions [UIApplication.LaunchOptionsKey Any]) - Bool {
        if WCSession.isSupported() {
            let session = WCSession.default
            session.delegate = self
            session.activate()
        }
        return true
    }
}
```

Шаг 2. Отправка данных с iPhone на Watch

При созданииредактировании тренировки на iPhone отправляем её на Watch

```swift
func sendWorkoutToWatch(_ workout Workout) {
    guard WCSession.default.isReachable else {
         Ставим в очередь для отправки позже
        WCSession.default.transferUserInfo([workout workout.dictionaryRepresentation()])
        return
    }
    
     Мгновенная отправка
    WCSession.default.updateApplicationContext([currentWorkout workout.dictionaryRepresentation()])
}
```

Шаг 3. Приём данных на Watch

На Apple Watch обрабатываем полученные данные

```swift
extension ExtensionDelegate WCSessionDelegate {
    func session(_ session WCSession, didReceiveApplicationContext applicationContext [String  Any]) {
        if let workoutDict = applicationContext[currentWorkout] as [String Any] {
             Обновляем локальную базу данных
            saveWorkoutFromDictionary(workoutDict)
             Перерисовываем UI
            updateWorkoutList()
        }
    }
}
```

### 4. Нефункциональные требования

 Производительность приложение должно работать плавно даже на старых моделях iPhone и Apple Watch.
 Энергопотребление минимизировать расход батареи на Apple Watch во время тренировки (оптимизировать обновления UI, использовать энергоэффективные API).
 Доступность поддержка VoiceOver, Dynamic Type, контрастных тем.
 Локализация предусмотреть возможность перевода интерфейса на другие языки (начать с русского и английского).
 Обработка ошибок
     проверка `isReachable` перед отправкой данных;
     сохранение неотправленных данных локально и повторная отправка при восстановлении соединения;
     уведомление пользователя, если синхронизация недоступна.
 Оптимизация
     отправка только изменений (дифф), а не всего списка тренировок;
     группировка мелких обновлений в один `applicationContext`.
 UI‑фидбек
     на iPhone индикатор «Синхронизация…» при отправке данных;
     на Watch сообщение «iPhone не найден. Данные обновятся при подключении».

### 5. Формат данных

Используйте `Codable` для сериализации тренировок в `[String Any]` или JSON. Пример структуры

```swift
struct Workout Codable {
    let id UUID
    let name String
    var exercises [Exercise]
    var rounds Int  1 для бесконечного
}

struct Exercise Codable {
    let id UUID
    let name String
    let duration Int  в секундах
}
```

### 6. Сценарии использования

1. Создание тренировки на iPhone
     пользователь создаёт тренировку, добавляет упражнения, задаёт время и круги;
     данные сохраняются локально;
     приложение пытается отправить данные на Watch через WatchConnectivity;
     если Watch недоступен, данные ставятся в очередь и отправляются позже.
2. Запуск тренировки на Watch
     пользователь выбирает тренировку из списка;
     нажимает «Начать»;
     запускается таймер с обратной связью (вибрация, звук).
3. Редактирование тренировки
     изменения на iPhone синхронизируются с Watch при подключении.
